# Download-Based Installer Design

## Problem

The current installer uses `git clone` to copy the entire development repository into the user's workspace (`~/.cccbot`). This means:

- The user's workspace IS a clone of the dev repo (shared git history, remote pointing to dev repo)
- `git log` shows development commits, not the user's changes
- No clean separation between "CCCBot the tool" and "the user's workspace"

## Solution

Replace `git clone` with GitHub Releases archive download. After extraction, `git init` creates an independent repository for the user's workspace.

## Installation Flow

```
1. Check dependencies (git, claude CLI)
2. Resolve version (argument → specific tag, default → latest release via GitHub API)
3. Download source archive to isolated temp directory (tar.gz on Linux/macOS, zip on Windows)
4. Extract archive in temp directory, move contents to INSTALL_DIR
5. Interactive permission mode selection (bypass / allowEdits)
6. Place .gitignore from template
7. Copy settings.json.default → .claude/settings.json, replace defaultMode with selected permission
8. git init + initial commit (clean history, no remote)
9. Generate user config files from templates (only if missing)
10. Launch via start.sh / start.bat
```

## Version Resolution

- **Default**: Query GitHub API for latest release tag
  - Linux/macOS: `curl -s https://api.github.com/repos/lucianlamp/CCCBot/releases/latest | grep tag_name`
  - Windows: `(Invoke-WebRequest ...).Content | ConvertFrom-Json | Select tag_name`
- **Explicit**: `install.sh v1.0.0` or `install.bat v1.0.0` pins to a specific version
- URL pattern: `https://github.com/lucianlamp/CCCBot/archive/refs/tags/{tag}.tar.gz` (or `.zip`)

## Archive Handling

### Linux/macOS (install.sh)
```bash
TMPDIR=$(mktemp -d)
curl -fsSL "https://github.com/lucianlamp/CCCBot/archive/refs/tags/${VERSION}.tar.gz" -o "$TMPDIR/cccbot.tar.gz"
tar xzf "$TMPDIR/cccbot.tar.gz" -C "$TMPDIR"
# Archive extracts to CCCBot-{version}/ (GitHub strips leading 'v' from tag)
mkdir -p "$INSTALL_DIR"
mv "$TMPDIR"/CCCBot-*/* "$INSTALL_DIR/"
rm -rf "$TMPDIR"
```

### Windows (install.bat)
```bat
set "TMPDIR=%TEMP%\cccbot-%RANDOM%"
mkdir "%TMPDIR%"
powershell -NoProfile -Command "Invoke-WebRequest 'https://github.com/lucianlamp/CCCBot/archive/refs/tags/%VERSION%.zip' -OutFile '%TMPDIR%\cccbot.zip'"
powershell -NoProfile -Command "Expand-Archive -Force '%TMPDIR%\cccbot.zip' '%TMPDIR%\extract'"
rem Move contents from extracted CCCBot-{version}\ to INSTALL_DIR
if not exist "%INSTALL_DIR%" mkdir "%INSTALL_DIR%"
for /d %%d in ("%TMPDIR%\extract\CCCBot-*") do xcopy "%%d\*" "%INSTALL_DIR%\" /E /Y /Q
rd /s /q "%TMPDIR%"
```

## Permission Mode & Settings

After extraction, before `git init`:
1. Prompt user for permission mode (bypass / allowEdits)
2. Copy `scripts/templates/settings.json.default` → `.claude/settings.json`
3. Replace `"defaultMode": "bypassPermissions"` with selected mode
   - Linux/macOS: `sed -i`
   - Windows: PowerShell `-replace`

Only on fresh install — if `.claude/settings.json` already exists (update), skip.

## Git Init (Independent Workspace)

After file extraction and config setup:
```bash
git init
git add -A
git commit -m "CCCBot ${VERSION} installed"
```

- No remote configured — workspace is fully independent from dev repo
- User's subsequent changes (SOUL.md edits, CLAUDE.md tweaks) are tracked in their own history
- `.gitignore` placed before `git add -A` to exclude secrets/runtime files

### Migration from git-clone workspace

If the installer detects an existing `.git` with a remote pointing to the CCCBot dev repo (`lucianlamp/CCCBot`), it should:

1. Warn the user that the workspace was previously a clone
2. Remove the remote (`git remote remove origin`)
3. Continue with the update flow (overwrite core files, preserve user config, commit)

## User Config Files

### Generated by installer (from templates, only if missing)
- `CLAUDE.md` ← `CLAUDE.example.md`
- `BOOT.md` ← `BOOT.example.md`
- `HEARTBEAT.md` ← `HEARTBEAT.example.md`
- `JOBS.yaml` ← `JOBS.example.yaml`

### Generated at runtime (not by installer)
- `SOUL.md` — created by the `ccc-soul` skill on first boot via interactive Telegram setup
- `memory/`, `MEMORY.md` — created by auto-memory system during sessions

## Update Flow

Re-running the installer on an existing directory:

1. Download new version archive to isolated temp directory
2. Overwrite core files (skills, scripts, templates, launchers, README)
3. Preserve user config files — skip if exists
4. Handle legacy migration (git-clone workspace → remove remote)
5. `git add -A && git commit -m "CCCBot updated to ${VERSION}"` to record the update

> **Note:** Legacy `CRONS.md` → `JOBS.yaml` migration is handled by the `ccc-boot` skill at session start, not by the installer.

### Files overwritten on update (core)

- `.claude/skills/**`
- `scripts/**`
- `start.sh`, `start.bat`
- `README.md`, `README.ja.md`
- `LICENSE`

### Files preserved on update (user config)

- `CLAUDE.md`, `SOUL.md`, `BOOT.md`, `HEARTBEAT.md`, `JOBS.yaml`
- `.claude/settings.json`, `.claude/settings.local.json`
- `.mcp.json`
- `.gitignore` (user may have customized)
- `memory/` (gitignored, not tracked)

## Error Recovery

- Archive is downloaded and extracted in an isolated temp directory (`mktemp -d` / `%TEMP%\cccbot-%RANDOM%`)
- Files are moved to `INSTALL_DIR` only after successful extraction
- If download or extraction fails, temp directory is cleaned up and `INSTALL_DIR` remains unchanged
- The `git commit` step at the end records the final state, so partial updates are visible in `git status`

## Installer Distribution

The installer scripts live in `scripts/install.sh` and `scripts/install.bat` in the dev repo. Users execute them via raw URL one-liner:

```bash
# macOS / Linux
bash <(curl -fsSL https://raw.githubusercontent.com/lucianlamp/CCCBot/master/scripts/install.sh)

# With version pinning
bash <(curl -fsSL https://raw.githubusercontent.com/lucianlamp/CCCBot/master/scripts/install.sh) v1.0.0
```

```powershell
# Windows
$f="$env:TEMP\cccbot-install.bat"; (Invoke-WebRequest https://raw.githubusercontent.com/lucianlamp/CCCBot/master/scripts/install.bat).Content | Set-Content -Encoding ASCII $f; & $f
```

> **Note:** The one-liner always fetches the installer from `master`. This is acceptable because the installer's only role is to download the specified (or latest) release archive — it does not contain workspace logic itself.

## Release Workflow

Developers create releases via `gh release create` or GitHub UI:

1. Tag the commit (e.g., `v1.0.0`)
2. Create GitHub Release — Source code archives are auto-generated
3. Users running `install.sh` without arguments get this latest release

No CI/CD or custom asset build required.
